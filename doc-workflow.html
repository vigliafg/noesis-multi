📦 Architettura Unificata — noesis816-full.html

noesis816-full.html è la versione unificata di Noesis che combina Reader + Editor in un unico file HTML completamente offline (~1.78 MB). Tutte le dipendenze (bootstrap-icons, JSZip, epub.js, Summernote, Excalidraw) sono embedded inline. Zero richieste HTTP.

noesis816-full.html
├── 📚 LIBRARY VIEW        — Libreria gerarchica (libri → capitoli → snapshot)
├── 📖 READER VIEW         — Reader EPUB con TOC, temi, bookmarks, annotate
└── ✏️ EDITOR (embedded)   — Full Noesis Editor via sn56Source → _openSn56 → nuova finestra

Componenti chiave:
├── sn56Source             — JSON con l'intero HTML dell'editor (900KB embedded)
├── _openSn56(payload)     — Apre editor in nuova finestra con contenuto opzionale
├── libEditorBtn           — Pulsante EDITOR nella header della libreria
├── extractOpenEditor      — Voce "Open in Editor" nel dropdown Extract
├── IDB bridge (postMessage)— Comunicazione IndexedDB tra main app e finestra editor
└── Payload boot script    — Carica automaticamente il contenuto del capitolo nell'editor

Fase 5 — EDITOR (sn56 via _openSn56)

_openSn56(payload) — Flusso tecnico:

  1. window.open('', '_blank')           ← sincrono, evita popup blocker
  2. JSON.parse(sn56Source.textContent)   ← decodifica HTML editor
  3. Se payload:
     ├── Inietta <script id="noesisPayload"> con JSON del capitolo
     └── Inietta boot script che:
           ├── Ascolta DOMContentLoaded
           ├── Legge #noesisPayload
           ├── Trova .note-editing-area .note-editable
           └── Imposta innerHTML = payload.html
  4. w.document.write(sourceHtml)         ← scrive HTML nella finestra
  5. w.document.close()                   ← completa rendering

  Editor opened:
  ├── Modalità "chapter":  contenuto pre-caricato, contesto bibliografico
  └── Modalità "standalone": editor vuoto (_openSn56(null))

  Editor features (noesis816-full-editor):
  ├── Toolbar Summernote completa (H1-H6, bold, italic, liste, tabelle)
  ├── Inspect Panel — raccolta chunk (testo, immagini, tabelle)
  ├── Snapshot System — doppio salvataggio (IDB + filesystem)
  ├── Excalidraw integrato — diagrammi e flowchart
  └── Export: TXT, MD, MD+ZIP, JSON, DOCX, PDF

Architettura sn56Source

sn56Source è l'editor HTML completo embedded come JSON in un tag <script>:

<!-- SN56_SOURCE_START -->
<script type="application/json" id="sn56Source">
  "<!DOCTYPE html>...intero HTML dell'editor (900KB)...</html>"
</script>
<!-- SN56_SOURCE_END -->

Caratteristiche:
├── JSON-encoded: tutti i caratteri speciali escaped
├── </ escaped come <\\/ per evitare chiusura prematura del tag script
├── Contiene SN56_PAYLOAD_SLOT per iniezione contenuto capitolo
├── 100% offline: font, CSS, JS tutto inline (base64)
└── ~900KB JSON → ~750KB HTML editor effettivo

Il marker SN56_PAYLOAD_SLOT:
  Posizionato subito dopo <body> nell'editor HTML.
  _openSn56 lo sostituisce con:
    <script id="noesisPayload">{chapterId, html, title}</script>
    <script>...boot loader...</script>

IDB Bridge — postMessage

L'editor (in finestra separata) NON ha accesso diretto a IndexedDB
perché il DB è stato creato dalla main app. Il bridge risolve questo:

  FINESTRA EDITOR                    MAIN APP (noesis816-full.html)
  ──────────────                    ──────────────────────────────
  postMessage({                     window.addEventListener('message', e => {
    __noesisIDB: 'get',       →       // Apre NOESIS_DB_NAME/NOESIS_STORE
    id: chapterId                      // Esegue get/put/delete
  }, '*')                              // Risponde con:
                                 ←     postMessage({
                                         __noesisIDBResponse: true,
                                         id, result, error
                                       }, '*')

Operazioni supportate:
├── __noesisIDB: 'get'    → store.get(id)
├── __noesisIDB: 'put'    → store.put({id, data})
└── __noesisIDB: 'delete' → store.delete(id)

Database usato: NOESIS_DB_NAME ('noesisDB') / NOESIS_STORE ('extractedChapters')

Extract→Editor Flow

Flusso completo Lettura → Estrazione → Editing:

  1. Utente legge nel Reader
  2. Trova contenuto rilevante → evidenzia (contextual popup)
  3. Clicca Extract ▾ → "Open current chapter in Editor"
  4. extractCurrentChapter() viene chiamata:
     ├── Determina capitolo corrente (da rendition.currentLocation)
     ├── Carica HTML del capitolo dall'archivio EPUB
     ├── Converte immagini in base64 inline
     ├── Estrae CSS (inline + linked stylesheets)
     ├── Salva in IndexedDB (noesisDB)
     ├── Auto-download file HTML su disco
     └── RETURN {title, chapterId, html: htmlContent, css: cssContent}
  5. _openSn56(payload) viene chiamata con i dati del capitolo
  6. Nuova finestra si apre con l'editor e il contenuto già caricato
  7. Utente modifica, aggiunge note, esporta

  Tempo totale: ~3-5 secondi dall'click all'editor pronto

Sistema Snapshot

SNAPSHOT — Doppio salvataggio (IDB + Filesystem):

  Save Snapshot (pulsante 📷 nell'Editor):
  ├── Cattura innerHTML corrente
  ├── Salva in IndexedDB (noesisDB → chapterRecord.snapshots[])
  └── Download automatico su disco:
        ├── noesis-clean-*.html  (senza colori di sfondo)
        └── noesis-annot-*.html  (con highlight preservati)

  Snapshot History (pulsante 🕐 nell'Editor):
  ├── Lista cronologica (dal più recente)
  ├── [↩ Load] → ripristina contenuto a quello snapshot
  └── [🗑️ Delete] → elimina snapshot

  Accesso dalla Libreria:
  ├── Click nome capitolo → snapshot più recente
  └── Click nome snapshot → quello specifico

Export — Schema riassuntivo

EXPORT DALL'EDITOR:

  Documento:
  ├── TXT     — Testo puro
  ├── MD      — Markdown (via Turndown)
  ├── MD+ZIP  — Markdown + immagini in ZIP
  ├── JSON    — Struttura documento
  ├── DOCX    — Word (via html-docx-js)
  └── PDF     — Stampa nativa browser (@media print A4)

  Collezione Chunk:
  ├── JSON    — Array chunk importabile/esportabile
  ├── HTML    — Chunk come documento standalone
  ├── MD      — Chunk in Markdown
  └── MD+ZIP  — Chunk Markdown + immagini ZIP

  Snapshot (filesystem):
  ├── noesis-clean-*.html   (senza highlight)
  ├── noesis-annot-*.html   (con highlight)
  └── noesis-origin-*.html  (reimportabile via meta tag)

Persistenza dei dati

COSA VIENE SALVATO E DOVE:

  localStorage:
  ├── Preferenze tipografia (font size, interlinea, page width)
  ├── Tema di lettura selezionato (15 temi)
  ├── Impostazioni interfaccia (colori toolbar/sidebar, opacità)
  ├── Posizione di lettura per ogni libro (CFI)
  └── Highlights (CFI-based, per libro)

  IndexedDB — EpubLibraryDB:
  ├── Libri importati (file .epub completi come ArrayBuffer)
  ├── Metadati libro (titolo, autore, copertina)
  └── Segnalibri per ogni libro

  IndexedDB — noesisDB (NOESIS_DB_NAME):
  ├── Capitoli estratti (chapterRecord)
  │     { chapterId, bookName, chapterName, createdAt, snapshots[] }
  └── Snapshot per capitolo
        { snapshotId, createdAt, description, content (HTML), isOrigin }

Note operative

  • Popup blocker: _openSn56 usa window.open() sincrono nel click handler per evitare il blocco. Se il popup viene bloccato, appare un alert.
  • about:blank origin: La finestra editor viene aperta con window.open('', '_blank') e popolata via document.write(). L'editor è completamente self-contained (zero risorse esterne), quindi l'assenza di origin non causa problemi.
  • sn56Source size: ~900KB JSON → ~1.78MB file totale. Accettabile per una "full" version offline.
  • CORS: Per alcuni EPUB, avviare un server locale (python -m http.server 8765) invece di file://.
  • Backup: Esportare periodicamente le collezioni come JSON. Gli snapshot sono già salvati su disco + IDB.
  • Manutenzione: Per aggiornare l'editor embedded, modificare noesis816-full-editor.html e rieseguire il merge script.
