Tre punti di accesso dalla libreria:

CLICK SU COPERTINA        → Apre il libro nel Reader
                             (ripristina ultima posizione)

CLICK SU NOME CAPITOLO    → Apre il capitolo estratto
                             con lo snapshot più recente

CLICK SU NOME SNAPSHOT    → Apre il capitolo estratto
                             con quello snapshot specifico

Azioni disponibili:
- Importare un nuovo file .epub
- Aprire un libro già importato (salvato in IndexedDB)
- Accedere direttamente a un capitolo estratto con lo snapshot desiderato
- Eliminare un libro dalla libreria

Fase 3b — Raccolta Chunk (sn56.x)

Durante la lettura del documento estratto in sn56.x, è possibile raccogliere frammenti di testo, immagini e tabelle in una collezione:

COLLEZIONE
  │
  ├── ADD SELECTION
  │     ├── Seleziona testo → click [+]
  │     ├── Doppio tap su immagine
  │     ├── Long press (600ms) su immagine (Android)
  │     └── Click [+ su tabella]
  │
  ├── IMPORT COLLECTION
  │     └── Carica file .json con collezione precedente
  │
  └── EXPORT COLLECTION
        └── Salva collezione corrente come file .json
              (portabile, archiviabile, condivisibile)

Metodi di raccolta:

  • Pulsante [+]: aggiunge testo selezionato, immagine tracciata, o tabella
  • Doppio tap su immagine: aggiunge direttamente
  • Long press (600ms) su immagine: aggiunge direttamente (Android)

Gestione avanzata: Inspect Panel per riordinare, iniettare al cursore, o assemblare tutti i chunk nel documento.

Fase 3c — Sistema Snapshot doppio (IDB + Filesystem)

Il sistema snapshot salva lo stato del documento sia in IndexedDB (per accesso rapido dalla Library) che su filesystem (backup persistente).

Salvataggio snapshot:

  1. Click su Save nell'Editor
  2. Opzionale: inserire etichetta personalizzata
  3. Generazione automatica di due file scaricati su disco:
    • noesis-clean-...html — documento senza background-color (evidenziazioni rimosse)
    • noesis-annot-...html — documento con highlight inline preservati
  4. Contemporaneamente: salvataggio di due snapshot in IndexedDB (noesisDB)
SNAPSHOT WORKFLOW
  │
  ├── SAVE SNAPSHOT (pulsante 📷 viola nella toolbar)
  │     │
  │     ├── Click → dialogo "Save Snapshot"
  │     ├── Inserisci descrizione (opzionale)
  │     │     Es.: "Versione base", "Dopo highlight", "Revisione 2"
  │     ├── Conferma → cattura innerHTML di .chapter-content
  │     └── Salva in IndexedDB (noesisDB)
  │           → toast "✅ Snapshot saved"
  │
  ├── SNAPSHOT HISTORY (pulsante 🕐 viola nella toolbar)
  │     │
  │     ├── Click → pannello cronologico (dal più recente)
  │     │
  │     ├── Per ogni snapshot:
  │     │     ├── Descrizione + data/ora
  │     │     ├── [↩ Load]  → ripristina il contenuto del capitolo
  │     │     │               allo stato di quello snapshot
  │     │     └── [🗑️ Delete] → elimina definitivamente lo snapshot
  │     │
  │     └── Stato vuoto: "No snapshots yet."
  │
  └── ACCESSO DALLA LIBRERIA
        ├── Click nome capitolo → carica snapshot più recente
        └── Click nome snapshot → carica quello snapshot specifico

Scenari tipici di utilizzo degli snapshot:

SCENARIO STUDIO PROGRESSIVO:
  1. Estrai capitolo (snapshot automaticamente disponibile al salvataggio)
  2. Leggi e annota → Salva snapshot "Lettura iniziale"
  3. Aggiungi highlight → Salva snapshot "Con highlights"
  4. Raccogli chunk → Salva snapshot "Dopo raccolta"
  5. Dalla libreria: accedi a qualsiasi versione con un click

SCENARIO BACKUP PRIMA DI MODIFICHE:
  1. Hai un capitolo ben annotato
  2. Salva snapshot "Versione definitiva" prima di procedere
  3. Modifica liberamente
  4. Se necessario, ripristina da Snapshot History

SCENARIO COMPARAZIONE VERSIONI:
  1. Salva più snapshot con descrizioni esplicite
  2. Apri versioni diverse dalla libreria in tab separate
  3. Confronta le annotazioni nelle diverse fasi di studio

Fase 4 — EDITOR WYSIWYG (sn56.x / Noesis Editor)

Produzione del documento finale. L'editor WYSIWYG è l'ambiente di annotazione e studio di Noesis, basato su Summernote-lite 0.9.1. È noto anche come sn56.x o Summernote Editor.

Nota: L'ambiente Extract Chapter è ora unificato con il Noesis Editor (sn56.x). Quando estrai un capitolo dal Reader, si apre direttamente nell'Editor.

Modalità operative

Modalità Attivazione Caratteristiche
chapter Da Reader (Extract) o Library (click snapshot) Caricato con HTML del capitolo, contesto bibliografico (bookName, chapterName, chapterId), salvataggio in IDB + filesystem
standalone Library → "Open Editor" Editor vuoto, nessun contesto, salvataggio solo su filesystem

Tre percorsi di accesso

  1. Library → Open Editor: apre editor vuoto (modalità standalone)
  2. Library → click nome capitolo/snapshot: apre con snapshot più recente o specifico (modalità chapter)
  3. Reader → Extract: estrae capitolo e apre immediatamente in editor (modalità chapter)

Export — Schema riassuntivo

DOVE POSSO ESPORTARE?

Dall'Noesis Editor — Documento:
  ├── TXT    — Testo puro
  ├── MD     — Markdown (via Turndown)
  ├── MD+ZIP — Markdown + immagini estratte in ZIP
  ├── JSON   — Struttura documento come JSON
  ├── DOCX   — Word (via html-docx-js)
  └── PDF    — Stampa nativa browser (@media print A4)

Dall'Noesis Editor — Collezione Chunk:
  ├── JSON   — Array chunk (importabile)
  ├── HTML   — Chunk come documento standalone
  ├── MD     — Chunk in Markdown
  └── MD+ZIP — Chunk in Markdown + immagini ZIP

Snapshot (su filesystem locale):
  ├── 📸  noesis-clean-...html  (senza highlight)
  ├── 📸  noesis-annot-...html  (con highlight)
  └── 📸  noesis-origin-...html (reimportabile via meta tag)
          → Accessibili dalla libreria gerarchica via Import Snapshots

Workflow per profilo utente

🎓 Studente — Workflow rapido

1. Importa il manuale (.epub)
2. Naviga al capitolo da studiare (TOC sidebar)
3. Leggi + evidenzia concetti chiave (giallo)
4. Evidenzia esempi (verde) e date/nomi (rosa)
5. Estrai il capitolo (corrente + sottolivelli)
6. Salva snapshot "Lettura base"
7. Aggiungi selezioni importanti alla collezione
8. Salva snapshot "Dopo raccolta chunk"
9. Lancia l'editor dalla collezione
10. Struttura gli appunti: H1/H2, liste
11. Esporta in PDF → consegna o stampa
12. Riapri versioni precedenti dalla libreria se necessario

🔬 Ricercatore — Workflow approfondito

1. Importa il paper/monografia (.epub)
2. Leggi con traduzione browser attiva se lingua straniera
3. Seleziona grafici e immagini → aggiungi alla collezione (base64)
4. Seleziona citazioni rilevanti → collezione
5. Salva snapshot "Prima revisione"
6. Esporta la collezione come JSON (backup)
7. Continua l'analisi, aggiungi annotazioni
8. Salva snapshot "Annotazioni complete"
9. Lancia editor dalla collezione
10. Struttura il report: titoli, tabelle, immagini
11. Esporta in DOCX per revisione collegiale

📝 Blogger/Creator — Workflow veloce

1. Leggi il libro con Noesis
2. Seleziona i passaggi per la review → collezione
3. Estrai i capitoli rilevanti
4. Salva snapshot "Materiale raccolto"
5. Lancia editor dal capitolo estratto
6. Scrivi la review nell'editor WYSIWYG
7. Copia HTML → incolla nel CMS (WordPress, Ghost, ecc.)

🌍 Lettore multilingue — Workflow studio lingua

1. Importa romanzo o testo nella lingua target
2. Attiva traduzione browser per lingua madre
3. Leggi con doppio riferimento (originale + traduzione)
4. Highlight parole nuove (rosa)
5. Estrai il capitolo → Salva snapshot "Vocabolario sessione 1"
7. Collezione delle parole/frasi difficili
8. Esporta collezione JSON → archivio personale vocabolario

Persistenza dei dati

COSA VIENE SALVATO E DOVE

localStorage:
  ├── Preferenze tipografia (font size, interlinea)
  ├── Tema di lettura selezionato
  └── Impostazioni interfaccia (colori, opacità)

IndexedDB — EpubLibraryDB:
  ├── Libri importati (file .epub completi come ArrayBuffer)
  ├── Posizione di lettura per ogni libro
  └── Segnalibri per ogni libro

IndexedDB — noesisDB  (NUOVO):
  ├── Capitoli estratti (chapterRecord con chapterId, bookName, chapterName, createdAt)
  └── Snapshot per ogni capitolo (array all'interno del chapterRecord)
        Ogni snapshot: { snapshotId, createdAt, bookName, chapterName,
                         description, content (HTML completo) }

File system (export):
  ├── .json → collezioni esportate
  ├── .pdf  → documenti esportati
  └── .docx → documenti Word esportati

Note operative

  • CORS: Se alcuni libri non si rendono correttamente, avvia un server locale (python -m http.server 8000) invece di aprire il file con file://
  • Offline: Dopo il primo caricamento (scarica CDN), Noesis funziona completamente offline
  • Backup: Esporta periodicamente le collezioni come JSON per non perdere il lavoro di annotazione
  • Snapshot: Salva snapshot prima di modifiche importanti — è il modo più sicuro per tornare a una versione precedente del capitolo estratto
  • Immagini nell'editor: Vengono salvate inline come base64 — il documento è sempre autocontenuto, senza path rotti
  • Libreria: Al riavvio dell'app, la libreria mostra automaticamente tutti i capitoli estratti con i loro snapshot, senza necessità di riaprire i file
