📚 LIBRARY — Libreria Gerarchica

L'ambiente di ingresso. Mostra tutti i libri importati con la loro gerarchia completa di capitoli estratti e snapshot. Il tema dell'interfaccia è scuro con tipografia editoriale.

LIBRARY
│
├── ── HEADER ──────────────────────────────────────────────
│
├── [Themes ▾]  Selezione Tema (dropdown)
│         Icona:    bi-sun-moon
│         Funzione: Apre un dropdown con due voci: Light e Dark.
│                   La selezione applica il tema immediatamente.
│                   Preferenza salvata in localStorage.
│         Trigger:  Click → apre dropdown; click voce → applica tema
│
├── [+]  Add Book
│         Icona:    bi-plus-lg
│         Funzione: Apre il file picker del sistema operativo.
│                   Accetta file .epub. Il libro viene letto
│                   come ArrayBuffer e salvato in IndexedDB
│                   (EpubLibraryDB). Appare immediatamente
│                   nella lista libreria.
│         Trigger:  Click → dialog di selezione file → conferma
│
├── [↑]  Import Snapshots
│         Icona:    bi-cloud-upload
│         Funzione: Reimporta file snapshot HTML precedentemente
│                   salvati su disco in noesisDB.
│                   Su Chrome/Edge: showDirectoryPicker().
│                   Su Safari/Android: file picker multi-file.
│                   Riconosce i file tramite meta tag noesis-*.
│         Trigger:  Click → selezione file/cartella → parsing meta tag
│
├── [✏️]  Open Editor
│         Funzione: Apre in modalità standalone (editor vuoto,
│                   nessun libro associato).
│         Trigger:  Click → Blob URL → nuovo tab Editor
│
├── [🛠️]  Tools ▾
│         Funzione: Dropdown con strumenti esterni:
│         ├── noesis-epub-tools  — Editor e gestione avanzata EPUB
│         ├── Pandoc Online      — Convertitore formati via pandoc.wasm
│         │                        nel browser (MD, DOCX, HTML, LaTeX,
│         │                        EPUB, ODT). Nessun upload di file.
│         └── Mozilla PDF Viewer — Visualizzatore PDF basato su PDF.js
│
│
├── ── STRUTTURA GERARCHICA ─────────────────────────────────
│
│   Per ogni libro viene visualizzata una riga espansa:
│
├── BOOK ROW
│   │
│   ├── [📖]  Copertina libro  (click → apre Reader)
│   │         Icona:    Immagine di copertina o bi-book
│   │         Funzione: Carica il file EPUB dall'IndexedDB,
│   │                   ripristina l'ultima posizione di lettura
│   │                   salvata e apre l'ambiente Reader.
│   │         Trigger:  Click sulla copertina → transizione a Reader
│   │
│   ├── [testo]  Titolo libro
│   │             Stile:    Corsivo, colore chiaro su sfondo scuro
│   │             Funzione: Visualizzazione. Non è cliccabile
│   │                       (usare la copertina per aprire).
│   │
│   ├── [badge]  Contatori
│   │             Funzione: Mostra il numero di capitoli estratti
│   │                       e il numero totale di snapshot salvati
│   │                       per quel libro.
│   │             Es.:  "2 chapters"  "5 snapshots"
│   │
│   ├── [🗑️]  Delete Book
│   │         Icona:    bi-trash
│   │         Funzione: Rimuove il libro dall'IndexedDB del browser.
│   │         Trigger:  Click → conferma dialog
│   │
│   └── CAPITOLI ESTRATTI (sotto ogni libro)
│       │
│       │   Se non ci sono capitoli estratti:
│       │   └── "No extracted chapters yet." (nota grigia)
│       │
│       ├── CHAPTER ENTRY  (uno per ogni capitolo estratto)
│       │   │
│       │   ├── [testo]  Nome capitolo  (click → apre snapshot più recente)
│       │   │             Funzione: Apre il capitolo estratto nell'ambiente
│       │   │                       Extract Chapter, caricando automaticamente
│       │   │                       lo snapshot più recente disponibile.
│       │   │             Trigger:  Click → window.open() con contenuto snapshot
│       │   │
│       │   ├── [badge]  Contatore snapshot
│       │   │             Es.: "3 snapshots" (viola) oppure "no snapshots" (grigio)
│       │   │
│       │   └── SNAPSHOT LIST (sotto ogni capitolo, dal più recente)
│       │       │
│       │       ├── [●]  Snapshot item  (click → apre quello snapshot)
│       │       │         Icona:    Dot colorato (verde per il più recente,
│       │       │                   viola per i precedenti)
│       │       │         Funzione: Apre il capitolo estratto nell'ambiente
│       │       │                   Extract Chapter, ripristinando esattamente
│       │       │                   il contenuto HTML di quello snapshot.
│       │       │         Info:     Descrizione snapshot + data/ora
│       │       │         Trigger:  Click → window.open() con contenuto snapshot
│       │       │
│       │       └── [ordinamento]  Dal più recente al meno recente

📖 READER — Ambiente di Lettura

│         Funzione: In modalità paginata: va alla pagina precedente.
│                   In modalità scroll: va al capitolo precedente
│                   nella spine del libro.

      │

      │
      └── [🌿]  Corrente + tutti i sottolivelli
                Funzione: Estrae il capitolo corrente e ricorsivamente
                          tutti i suoi sottolivelli. Assembla tutto in un
                          unico documento HTML standalone.
                          Salva il chapterRecord in noesisDB.

✏️ NOESIS EDITOR — Editor WYSIWYG (sn56.x / Summernote Editor)

L'editor Noesis Editor (codename sn56.x, talvolta chiamato "Summernote Editor") è l'ambiente di annotazione e studio di Noesis. È un editor WYSIWYG HTML completo basato su Summernote-lite 0.9.1 che opera in una finestra separata (contesto blob:null).

Nota: L'ambiente Extract Chapter è stato unificato con Noesis Editor. Quando estrai un capitolo dal Reader, si apre direttamente nell'Editor con il contenuto del capitolo caricato.

Layout interfaccia

┌─────────────────────────────────────────────────────────────┐
│ HEADER (#0a84ff blu)                                        │
│ "Nome capitolo / Noesis Editor"              [?] Help      │
├─────────────────────────────────────────────────────────────┤
│ SUMMERNOTE TOOLBAR (generata da Summernote)                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│ AREA EDITOR WYSIWYG (.note-editable)                       │
│ (altezza dinamica, occupa tutto lo spazio disponibile)     │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│ BOTTOM TOOLBAR (#111 nero)                                 │
│ [CHAPTER] [COLLECTION] [TOOLS]                              │
└─────────────────────────────────────────────────────────────┘

Tre percorsi di accesso

Percorso Trigger Modalità
1. Library → Open Editor Click "Open Editor" nella toolbar standalone — editor vuoto, nessun libro associato
2. Library → capitolo/snapshot Click su nome capitolo o snapshot chapter — carica contenuto HTML, contesto bibliografico
3. Reader → Extract Click "Extract" nel Reader chapter — estrae e apre immediatamente

Modalità operative

Modalità Caratteristiche
chapter Contenuto HTML caricato, contesto bibliografico (bookName, chapterName, chapterId), salvataggio in IDB + filesystem
standalone Editor vuoto, nessun contesto, salvataggio solo su filesystem

Header

  • Colore: #0a84ff (blu iOS)
  • Titolo (#appHeaderTitle): in modalità chapter mostra chapterName (o bookName come fallback); in standalone mostra "Noesis Editor"
  • Pulsante ? (#editorHelpBtn): apre l'overlay di aiuto contestuale

Inspect Panel — Pannello Chunk

Pannello flottante non modale, draggable e resizable:

  • Lista chunk con anteprima
  • Selezione multipla checkbox
  • Azioni per chunk: Inject (al cursore), Fullscreen, Delete
  • Inject di gruppo per tutti i chunk selezionati

Sistema Snapshot (doppia memorizzazione)

Il pulsante Save genera quattro output simultanei:

# Destinazione File/Oggetto Tipo
1 Filesystem noesis-clean-…html HTML senza background-color
2 Filesystem noesis-annot-…html HTML con highlight preservati
3 IndexedDB snapshot annot-… JSON in IDB
4 IndexedDB snapshot clean-… JSON in IDB

Schema naming: noesis-{tipo}-{BOOK}__{CHAPTER}__{YYYYMMDD_HHMMSS}[_{label}].html
