📚 Noesis — Reading-to-Knowledge System

Leggi in qualsiasi lingua. Trasforma i libri in conoscenza.





Noesis è un EPUB reader poliglotta e un ambiente di studio integrato che gira interamente nel browser, senza server, senza account e senza installazioni. Non è solo un lettore: è un sistema per trasformare la lettura in produzione attiva di conoscenza.

Cos'è Noesis

Noesis nasce come risposta a una domanda precisa: perché il lettore digitale e gli strumenti di studio devono essere applicazioni separate?

L'applicazione unisce in un unico file HTML autocontenuto:

  • un reader EPUB completo con navigazione gerarchica dell'indice
  • un sistema di traduzione nativa via browser per leggere in qualsiasi lingua
  • un motore di evidenziazione e annotazione con raccolta chunk
  • un estrattore di capitoli intelligente che rispetta la gerarchia del libro
  • un sistema di versioning a snapshot per conservare lo stato dei capitoli estratti nel tempo
  • un editor WYSIWYG per produrre report, dispense e documenti
  • un sistema di export multiplo (PDF, DOCX, HTML, PDF viewer)
  • un sistema di segnalibri, collezioni e gestione della posizione di lettura
  • una libreria gerarchica che mostra libri, capitoli estratti e snapshot in un'unica vista

Tutto funziona offline, nel browser, senza mai inviare un byte a server esterni.

Funzionalità principali

📖 Lettura EPUB completa

  • Rendering di file .epub direttamente nel browser tramite ePub.js
  • Navigazione con TOC gerarchico espandibile in sidebar
  • Modalità scorrimento continuo o navigazione paginata
  • Supporto a libri con struttura annidata multilivello

🌍 Lettura multilingue

  • Traduzione istantanea tramite la funzione di traduzione nativa del browser
  • Nessuna API esterna richiesta, nessun costo per carattere tradotto
  • Lettura fluida di qualsiasi lingua: ideale per leggere bestseller stranieri prima della traduzione ufficiale

🎨 Highlighting a tre colori

  • Evidenziazione in giallo, verde e rosa
  • Rimozione selettiva delle evidenziazioni

📦 Raccolta chunk e collezioni

  • Selezione di testo e immagini durante la lettura → aggiunti alla collezione
  • Collezioni importabili/esportabili come file JSON
  • I chunk raccolti alimentano l'editor report

🗂️ Estrazione capitoli intelligente

  • Solo il capitolo corrente: documento HTML standalone focalizzato
  • Capitolo corrente + tutti i sottolivelli: assemblaggio automatico dell'intera sezione con gerarchia preservata
  • Ogni documento estratto include toolbar completa, evidenziazioni e opzioni di export già attive
  • Il capitolo estratto viene salvato automaticamente in IndexedDB (noesisDB) con un ID univoco

📸 Sistema di versioning a Snapshot

  • Save Snapshot: salva lo stato HTML completo del capitolo estratto con una descrizione libera
  • Snapshot History: elenco cronologico inverso di tutti gli snapshot salvati per quel capitolo
  • Ogni snapshot è ricaricabile con un click (il contenuto del capitolo viene ripristinato allo stato salvato)
  • Ogni snapshot è cancellabile individualmente
  • Gli snapshot vengono salvati come file HTML su filesystem locale (File System Access API) — i metadati in noesisDB, i file su disco

📚 Libreria gerarchica

  • Vista libreria completamente rinnovata con tema scuro editoriale
  • Per ogni libro: copertina, titolo, autore e badge con contatori di capitoli estratti e snapshot totali
  • Sotto ogni libro: elenco dei capitoli estratti con i relativi snapshot (dal più recente)
  • Tre punti di accesso rapido:
  • Click sulla copertina → apre il libro nel Reader con lo stato memorizzato
  • Click sul nome del capitolo → apre il capitolo estratto con lo snapshot più recente
  • Click sul nome dello snapshot → apre il capitolo estratto con quello snapshot specifico

✏️ Editor WYSIWYG integrato —

  • Editor Summernote 0.9.1 embedded, aperto in nuovo tab via Blob URL
  • Formattazione: grassetto, corsivo, sottolineato, barrato
  • Intestazioni: H1, H2, H3, paragrafo, liste puntate e numerate
  • Inserimento immagini da file locale (salvate inline come base64, documento autocontenuto)
  • Font size regolabile (8–28px)
  • Plugin tabelle con toolbar contestuale flottante: inserisci/elimina righe e colonne
  • Raccolta Chunk: seleziona testo, immagini e tabelle dal documento
  • Inspect Panel flottante per gestire e iniettare chunk nel documento
  • Excalidraw integrato (v810): diagrammi, flowchart e mappe mentali in tab dedicata; esporta SVG/PNG
  • Save Snapshot: scarica su disco file HTML clean + annot

📄 Export multiplo

  • Documento: TXT, MD, MD+ZIP, JSON, DOCX, PDF
  • Collezione chunk: JSON, HTML, MD, MD+ZIP
  • Le immagini sono sempre inline (base64) — nessun path rotto

🔖 Segnalibri personali

  • Cassetto segnalibri con badge contatore
  • Salvataggio manuale della posizione di lettura e delle preferenze per ogni libro

🎨 15 temi di lettura

  • 5 gruppi tematici per ogni condizione di luce: chiari, caldi, grigi, scuri, notturni
  • Anteprima visiva con swatches colorati

⚙️ Personalizzazione interfaccia

  • Font size e interlinea configurabili
  • Pagina singola o doppia
  • Colori toolbar, sidebar e pulsanti di navigazione personalizzabili
  • Opacità dei controlli regolabile
  • Zoom toolbar (90%–130%)

Installazione e avvio

Noesis non richiede installazione. È un singolo file HTML con tutte le dipendenze già incluse.

Metodo 1 — Apertura diretta (più semplice)

# 1. Scarica il file zip dal sito
#    https://noesis.vigliafg.workers.dev/#download

# 2. Decomprimi
# Windows: tasto destro → Estrai tutto…
# macOS:   doppio click sullo zip
# Linux:
unzip noesis810.zip

# 3. Apri nel browser
# Doppio click su noesis810.html
# oppure trascinalo in Chrome, Firefox o Edge

Metodo 2 — Server locale (consigliato)

Alcuni file EPUB applicano restrizioni CORS che impediscono il rendering corretto se il file HTML viene aperto con file://. Per evitare questo problema:

# Python 3 (consigliato)
cd cartella-noesis
python -m http.server 8000
# Apri: http://localhost:8000/noesis810.html

# Python 2
python -m SimpleHTTPServer 8000

# Node.js (se disponibile)
npx serve .
# oppure
npx http-server . -p 8000

⚠️ Nota: Noesis funziona interamente nel browser. I libri sono salvati in IndexedDB (EpubLibraryDB), i metadati dei capitoli estratti in IndexedDB (noesisDB), gli snapshot come file HTML su filesystem locale, le preferenze in localStorage. Nessun dato lascia il dispositivo.

Utilizzo

Importare il primo libro

  1. Apri noesis810.html nel browser
  2. Clicca + Add Book nella schermata libreria
  3. Seleziona un file .epub dal tuo dispositivo
  4. Il libro appare nella libreria e si apre automaticamente

Leggere un libro

  1. Clicca sulla copertina del libro in libreria
  2. Usa la sidebar TOC per navigare tra i capitoli
  3. Attiva la traduzione del browser per leggere in altre lingue
  4. Usa i controlli tipografia per personalizzare la visualizzazione

Evidenziare e raccogliere

  1. Seleziona il testo che vuoi evidenziare — appare un popup contestuale con 4 colori
  2. Clicca il colore nel popup per applicarlo subito (Giallo / Verde / Rosa / ✕ Rimuovi)
  3. Per raccogliere nella collezione: seleziona → Add Selection

Estrarre e studiare

  1. Naviga al capitolo desiderato
  2. Clicca Extract Chapter → scegli tra solo corrente o corrente + sottolivelli
  3. Il documento estratto si apre in una nuova vista con toolbar completa
  4. Highlights, chunk e opzioni di export sono già disponibili

Accedere ai capitoli dalla libreria

  1. Nella libreria, espandi un libro per vedere i suoi capitoli estratti
  2. Clicca sul nome del capitolo per aprirlo con lo snapshot più recente
  3. Clicca sul nome di uno snapshot specifico per aprirlo direttamente
  4. Clicca sulla copertina per aprire il libro nel reader

Creare un report

  1. Estrai un capitolo dal Reader — si apre direttamente in Noesis Editor (sn56.x)
  2. Lavora nell'editor WYSIWYG: formatta, inserisci tabelle, aggiungi immagini
  3. Raccogli chunk dalla collezione se necessario
  4. Salva snapshot per versionare il lavoro
  5. Esporta in PDF, DOCX, MD, oppure copia l'HTML per il CMS

Privacy e dati

Noesis è progettato con una filosofia zero server, zero tracking:

  • ✅ Tutti i libri salvati in IndexedDB (EpubLibraryDB) nel browser (locale)
  • ✅ Metadati capitoli estratti in IndexedDB (noesisDB), snapshot come file HTML su filesystem locale
  • ✅ Preferenze in localStorage (locale)
  • ✅ Nessuna comunicazione con server esterni
  • ✅ Funziona offline dopo il primo caricamento
  • ✅ Nessun account, nessun abbonamento
  • ✅ File singolo HTML — tutto autocontenuto

L'unica connessione esterna avviene al primo caricamento per scaricare le dipendenze da CDN (ePub.js, JSZip, Bootstrap Icons, jQuery, Summernote, Mozilla PDF.js). Dopo di che, l'app funziona completamente offline.

Stack tecnico

Noesis usa esclusivamente Vanilla JS senza framework. Le dipendenze sono limitate a librerie specifiche caricate da CDN:

Libreria Scopo Caricamento
ePub.js Rendering EPUB nel browser CDN
JSZip Decompressione file EPUB CDN
Bootstrap Icons Iconografia dell'interfaccia CDN
jQuery Dipendenza di Summernote CDN
Summernote Editor WYSIWYG (Noesis Editor, Noesis fork) Built-in
Excalidraw Diagrammi e mindmap (Noesis fork) Built-in
Mozilla PDF.js Viewer PDF integrato CDN
pandoc.wasm Conversioni documento (Pandoc Online) Esterno (no upload)

Lo storage è gestito con due database IndexedDB separati: EpubLibraryDB per i libri e noesisDB per i capitoli estratti e gli snapshot.
