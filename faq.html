Support · FAQ

Frequently Asked Questions

Answers to the most common questions about Noesis — from installation to snapshots, from privacy to browser compatibility.

First launch

Installation and setup

There is nothing to install. Download noesis810.zip from the homepage, unzip it and double-click noesis810.html. The file opens in the browser and the app is ready.

For the best experience (especially with EPUBs that have external resources) start a local server: python -m http.server 8000 in the extracted folder, then open http://localhost:8000/noesis810.html.

Yes, Noesis is responsive and works on modern mobile browsers. The interface adapts to small screens with a hamburger menu for navigation. However, Noesis is primarily optimised for desktop use — document production (editor, export, snapshot management) is more comfortable on a computer with keyboard and large screen.

Only for the first load (downloads CDN libraries: ePub.js, JSZip, Bootstrap Icons, jQuery, Summernote, Mozilla PDF.js). After the first launch, Noesis works completely offline. Browser translation requires an internet connection (the translation service belongs to the browser, not Noesis).

Books and library

Import and book management

1. Verify the file is a valid EPUB and is not DRM-protected. Noesis does not support DRM files.

2. Use a local server. Open a shell in the Noesis folder and run python -m http.server 8000, then access http://localhost:8000/noesis810.html. Many EPUB loading issues are due to CORS restrictions with the file:// protocol.

3. Check IndexedDB in DevTools → Application → Storage → IndexedDB → EpubLibraryDB → books to verify the book was saved.

Technically unlimited — the library is only constrained by available IndexedDB space in the browser. In practice, modern browsers grant 50–100MB of IndexedDB storage by default (expandable). An average EPUB is 2–5MB, an EPUB with many images can reach 50–100MB. Use ePub Slimmer for large EPUBs.

Some EPUBs use non-standard relative paths for images — this can cause issues with the file:// protocol. Solutions:

  • Use the local server (python -m http.server 8000)
  • Extract the chapter: images are converted to inline base64 and are always visible in the extracted document
  • Use ePub Slimmer which rewrites image paths

Some EPUBs do not include a navigation file (NCX or nav.xhtml). You can verify by opening the browser console and checking book.navigation.toc. If it is empty or undefined, the book has no TOC and you can only navigate with the arrows.

Snapshots

Questions about snapshots

Yes. Snapshots are saved as HTML files on the local filesystem via the browser File System Access API. Metadata (name, date, file reference) is kept in noesisDB (IndexedDB). Files remain on disk until you explicitly delete them from the Snapshot History interface or from the filesystem.

Each snapshot contains the complete HTML of the chapter (no diff, no compression). An average chapter with text and some base64 images takes between 500KB and 5MB per snapshot. If you have many snapshots on chapters with many images, check space in DevTools → Application → Storage. Delete obsolete snapshots from Snapshot History.

Not automatically. Snapshots are HTML files on your local filesystem — there is no cloud or sync (by design — total privacy). To transfer work to another device: copy the snapshot files to the same folder on the new device, then import them from the Snapshot History interface. Also copy the EPUB file and the Noesis file.

If you had saved a snapshot of your current work before loading, you can simply reload that snapshot. If you had no snapshot of the current work, the restore is not undoable — the content has been replaced. Best practice: always save a snapshot before loading a previous version.

Export and Editor

Export, PDF and DOCX

In v810, PDF export uses the native browser print (window.print()) — no external libraries required and always works offline. If the print dialog does not open, verify that popups are not blocked. Alternatively, use the 👁 PDF Viewer button to open Mozilla PDF.js and print from there. For DOCX, use the DOCX button in the extracted chapter toolbar.

Yes. Images in the extracted chapter are already converted to inline base64. They are automatically included in PDF and DOCX export without additional configuration. The exported document is completely self-contained.

Use the </> Copy HTML button in the editor toolbar. It copies all document HTML to the clipboard. In WordPress, switch to HTML mode of the editor (HTML Code block or classic editor → text); in Ghost, use the HTML block; in Webflow, paste into a Rich Text or HTML Embed element.

The DOCX generated by Noesis follows the basic OOXML format. It works well in LibreOffice and Google Docs. In Word some complex formatting (nested tables, large base64 images) may be interpreted slightly differently. For optimal Word compatibility, use print → PDF as the final delivery format.

Privacy and storage

Data and privacy

All Noesis data is saved locally in your browser. No data leaves the device. The storage used:

Clearing browser data (Clear browsing data → Cookies and cached files → IndexedDB) removes books from the library, extracted chapter metadata and snapshot metadata from noesisDB. It does not delete snapshot HTML files that you have saved to the filesystem — those remain available for reimport. Books must be reimported from the original EPUB files.

Yes, completely. Noesis implements privacy by design: zero data transmission to external servers, zero tracking, zero analytics. All data remains in your browser and on your device. There is nothing to declare or manage from a GDPR perspective because no personal data is collected or transmitted.

Technology

Browser APIs used

💾

💾 IndexedDB

Persistent storage for books (EpubLibraryDB) and extracted chapters + snapshots (noesisDB). Asynchronous, supports large binary files.

📋

📋 Clipboard API

Captures text selections for chunks and copies editor HTML to the clipboard for pasting into a CMS.

🖼️

🖼️ Canvas API

Used for image compression and conversion in ePub Slimmer and for editor image resizing. PDF export in v810 uses native browser printing, not Canvas.

🌐

🌐 Translation API (browser)

Native integrated browser translation (Chrome, Edge, Firefox). This is not a Noesis API — it is a browser feature that Noesis does not control.

🖨️

🖨️ Print API

Native printing and PDF saving via window.print() with @media print CSS optimised for each context (reader, extracted chapter, editor).

📁

📁 FileReader API

Import of EPUB files and images from the user filesystem. Read as ArrayBuffer for JSZip and as base64 for editor images.
