Known issues

Limits and workarounds

OPEN

Android: File System Access API

On some Android versions, the File System Access API for saving snapshots may ask for permission each time. This is a browser limitation, not a Noesis bug. Workaround: use the Desktop version for intensive snapshot work.

OPEN

Safari: IndexedDB limitations

Safari imposes stricter storage limits for IndexedDB. Large EPUBs (>50MB) may fail to import. Workaround: use ePub Slimmer first to reduce size, or switch to Chrome/Firefox.

OPEN

file:// protocol: CORS restrictions

Opening Noesis directly with the file:// protocol can cause CORS issues with some EPUBs. Workaround: use a local server (python -m http.server 8000).

OPEN

DOCX and complex Word formatting

The DOCX generated by html-docx-js has basic OOXML formatting. Nested tables and very complex images may not render perfectly in Microsoft Word. Workaround: use PDF for delivery, DOCX for editing in LibreOffice/Google Docs.

Pro tips

Get the most from Noesis

💡 Workflow tip: snapshot before editing

Always save a snapshot before starting major editing or structural changes to a chapter. The 2-second investment protects against irreversible data loss.

💡 Multilingual reading: highlights stay on the original

Activate browser translation before reading a foreign text. Highlights are applied on the original text (CFI-based) and remain stable even when switching between original and translated view.

💡 Large EPUBs: slim before importing

If an EPUB has many high-resolution images (>20MB), use ePub Slimmer before importing. Significantly reduces loading time and space in IndexedDB.

💡 Export for CMS: use Copy HTML

For WordPress, Ghost or Webflow, use the </> Copy HTML button. The HTML is already clean with inline base64 images — paste directly into the CMS HTML block.
